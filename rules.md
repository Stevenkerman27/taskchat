开发规则：
1. GUI 包装器模式 (GUI Wrapper Architecture)
在开发具备图形界面的工具时，必须遵循“逻辑与表现分离”的原则，严禁将业务逻辑直接耦合在 GUI 事件回调中.

原则： GUI 仅仅是 CLI 的“输入生成器”与“结果展示器”.

所有核心功能必须首先实现为独立的 Command Line Interface (CLI) 或底层 API 模块。GUI 通过调用子进程与 CLI 交互，或通过统一的逻辑入口层进行调用。GUI 内部应存在一个显式的命令拼接逻辑。GUI必须直接显示终端信息并且允许输入。确保即便没有 GUI，程序依然可以通过脚本自动化运行.

2. DRY 原则 (Don't Repeat Yourself)
拒绝知识的重复。系统中的每一个功能点、算法或配置，都应有且仅有一个权威定义。禁止在多个地方手动同步相同的逻辑.

平衡点： 避免为了 DRY 而引入过度复杂的泛型或多层继承。如果消除重复会导致代码可读性急剧下降，请优先选择代码的清晰度，并辅助以显式注释.

3. 单一来源(Single Source of Truth)
常量、魔术字符串, 数据库 Schema 必须定义在集中配置文件中.

4. 注释规范
必须使用半角标点, 每个函数必须要简要的功能说明
