### 3. 工具调用流程
实现了完整的工具调用流程：

1. **配置加载**：启动时自动加载`tools.yaml`配置
2. **工具定义**：将工具配置转换为OpenAI格式的工具定义
3. **API调用**：发送包含工具定义的请求到DeepSeek API
4. **工具执行**：检测到工具调用后执行相应工具
5. **结果处理**：将工具结果添加到上下文并发送第二轮请求
6. **最终回复**：获取包含工具结果的最终回复

## 技术实现细节

### 工具定义格式
```python
{
    "type": "function",
    "function": {
        "name": "list_directory",
        "description": "列出当前目录下的文件和文件夹",
        "parameters": {
            "type": "object",
            "properties": {},
            "required": []
        }
    }
}
```

### 工具调用处理流程
1. 检测API响应中的`tool_calls`
2. 提取工具名称和参数
3. 调用`tools_impl.execute_tool()`执行工具
4. 将工具结果添加到消息上下文
5. 发送第二轮请求获取最终答案

- 支持动态启用/禁用工具

## 使用说明

### 启用工具调用
1. 启动ChatGUIV2应用程序
2. 在"Tool Management"区域确保"Enable Tools"已勾选
3. 启用的工具列表将显示在界面上

### 使用工具调用
1. 输入需要工具调用的请求，如"列出当前目录的文件"
2. 系统将自动检测并执行工具调用
3. 工具结果将自动处理并显示最终答案

### 禁用工具调用
1. 取消勾选"Enable Tools"
2. 工具定义将从API请求中移除
3. 系统将恢复为普通聊天模式
