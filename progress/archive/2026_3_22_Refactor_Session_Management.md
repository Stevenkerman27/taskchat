# 开发进展记录 - 2026-03-22

## 决策意图
1. **独立会话管理**：为了防止多个会话互相干扰，我们将原本单一的 `current_session.json` 重构为基于时间戳和随机后缀的独立会话文件（例如 `session-2026-03-22T13-57-af623d65.json`）。这样每次启动程序都会生成一个新的、不重复的历史记录，也便于用户追溯和并行运行多个实例。
2. **写时复制（Copy-on-Load）策略**：在加载历史会话时，不直接在原文件上修改，而是读取其内容后立即生成一个新的会话文件。这样保证了原始历史记录作为“快照”始终保持不变，任何新的对话都在新文件中进行，符合不可变历史的工程原则。
3. **指令语义重命名**：将 `/clear` 重命名为 `/new`。在语义上，`clear` 意味着“清空当前状态”，而 `new` 意味着“开启一个全新的篇章”。结合物理文件的变化，`new` 更能准确描述“切换到新文件并重置内存”的行为。
4. **GUI 精简**：移除了手动“保存”按钮。由于系统现在实现了每一轮对话的自动持久化（Auto-save to JSON），手动保存已无必要且会导致冗余。

## 防御性记录
- **路径连接 Bug**：在初步实现 `load_context_from_file` 时，发现如果传入的已经是包含目录的路径，再次 `os.path.join(self.contexts_dir, ...)` 会导致路径重复（如 `contexts/contexts/...`）。已通过增加路径判断逻辑修复，使其能智能识别相对路径、绝对路径以及已包含目录名的路径。
- **并发写入安全**：继续保留了原子写入（Atomic Write）机制（temp file -> os.replace），确保即使在自动保存过程中程序崩溃，也不会破坏已有的 JSON 数据。

## 验证依据
- **初始化验证**：通过 Python 脚本实例化 `ChatLogicV2`，确认其 `session_file` 每次都是唯一的，且成功在 `contexts` 目录下创建。
- **加载验证**：模拟加载操作，确认加载后 `session_file` 发生了变更（指向了新的独立文件），且成功恢复了历史消息。
- **CLI 验证**：确认 `/new` 指令和 `/help` 信息更新正确。
- **GUI 验证**：按钮重命名和功能映射（已在代码层确认逻辑一致性）。
