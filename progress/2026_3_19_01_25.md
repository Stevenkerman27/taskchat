- Modified `ChatGUIV2` in `chat_gui_v2.py` to fix the tool management layout.
- **Fixed Layout Issues**:
    - Changed `self.root.grid_rowconfigure(2, weight=3)` to `weight=0`. This prevents the Tool Management frame from expanding vertically and taking up disproportionate space.
    - Reallocated `weight=3` to `row 3` (Chat Output Area) and `weight=1` to `row 5` (Payload Preview Area) to ensure they expand properly when the window is resized.
- **Two-Row Tool Display**:
    - Updated `init_tool_groups_selection` to use a 2-row grid for tool group checkbuttons (`row = i % 2`, `col = i // 2`).

## Decision Intent
- The previous layout gave row 2 (Tools) a high weight, causing it to grow excessively when tool groups were loaded. By removing the weight and using a 2-row grid, the frame height remains stable and compact.
- Grid layout for checkbuttons is more manageable than packing them all in one row, especially as the number of tool groups grows.
